This paper, Generative Agents: Interactive Simulacra of Human Behavior, addresses the challenges of memory and continuity in generative agents by proposing an architecture that augments large language models with structured long-term memory, reflection, and retrieval mechanisms. Instead of relying on full conversation history, the system in the paper stores a stream of experiences in natural language, periodically compressing them into higher-level reflections, and retrieving only the most relevant memories based on recency, importance, and semantic relevance. This enables agents to maintain coherent behavior and evolving context over time while avoiding the computational cost of large prompts, supporting both continuity of experience and consistent personality. The paper shows how believable long-term behavior can come not from storing more data, but from organizing and filtering memory effectively.

 Not Everything Should Be Remembered: Making Generative Agents More Efficient and Human-Like
First, I’d like to briefly define what is meant by “generative agents” before diving into the topic. There are several types of agents, such as learning-based agents in reinforcement learning that optimize behavior for reward, task-oriented agents that complete user-driven tasks with some autonomy, and “believable agents” in cognitive models that simulate human behavior in open-world environments.
 
While all of these may benefit from memory and continuity, this discussion seems to focus specifically on agents that are “aware” of past experiences in order to maintain a consistent personality. Therefore, I will focus on the “believable agents” described in Generative Agents: Interactive Simulacra of Human Behavior.
 
In this work, Park et al. build an agent architecture on top of a large language model (LLM). This design has several advantages: memory, persona, and behavior can all be represented in natural language; agents can leverage pretrained knowledge to simulate human-like interactions; and the system is relatively interpretable since all internal states are readable as text. However, LLMs do not have intrinsic long-term memory, as their outputs are limited by the context window. This can lead to insufficient contextual reasoning and personality drift over time. Therefore, the key challenge is designing an external memory framework that enables continuity while maintaining efficiency.
 
The paper introduces several important components:
 
Memory stream
The memory stream is a chronological list that stores the agent’s observations and experiences. Memory retrieval is modeled as a function of recency, importance, and relevance, which are normalized into a score to select memories that fit within the current context. This allows the agent to balance short- and long-term memory when generating actions.
 
Reflection
Reflection allows the agent to periodically summarize and generalize past experiences into higher-level insights. This helps compress information and refine the agent’s persona over time, enabling more consistent behavior.
 
While this design is thoughtful and effective, it also feels somewhat deterministic and exhaustive compared to how human memory works. Based on this, I have a few ideas that might improve efficiency and realism:
 
1. Organizing memory hierarchically
The paper presents memory as a chronological stream, but conceptually it may be better represented as a graph or hierarchy. For example, repetitive routine behaviors (e.g., brushing teeth or daily activities) could be compressed into higher-level representations or habits instead of being stored as individual events. In contrast, important episodic events should remain as distinct memories with higher weight. This hierarchical compression would directly reduce the memory footprint and retrieval cost, addressing the efficiency constraint without sacrificing behavioral coherence.
 
2. Memory compression and bounded storage
Although reflection acts as a form of summarization, the paper does not explicitly limit the growth of the memory stream. A fixed-capacity memory system could be introduced, where older or less relevant memories are compressed into lower-dimensional representations. These compressed memories could later be reconstructed if needed, similar to representation learning. Some information loss may be acceptable as long as the core meaning is preserved. This is conceptually similar to how biological memory consolidation works — detailed episodic memories gradually compress into semantic knowledge, with only the gist preserved rather than verbatim experience.
 
3. Context-aware memory retrieval
The agent in the paper appears to retrieve and reason over memory for nearly every interaction. However, in real social settings, responses often depend on context and effort rather than strict relevance. For example, when asked a casual question like “what are you passionate about these days,” people often provide a quick, socially appropriate answer rather than deeply retrieving meaningful memories. This suggests that agents may benefit from a mechanism that detects interaction intent and adjusts the level of memory retrieval accordingly.
 
4. Introducing stochasticity and inter-agent variability in memory retrieval
The memory retrieval mechanism reminds me of sparse attention in transformers, where connections are limited to local, global, and random patterns. In this analogy, recent memories correspond to local attention, important memories to global tokens, and random recall to stochastic connections. Adding a small amount of randomness could prevent the system from being stuck in a narrow region of the memory space and allow more flexible, associative reasoning.
This also relates to personality consistency. Instead of personality being a fixed attribute, it could be viewed as a pattern of memory retrieval biases over time. For example, different agents may have different memory capacities or retrieval tendencies. Introducing variability across agents, along with occasional random recall, may improve realism while maintaining efficiency.
 
Overall, the challenge is not just storing memory, but deciding what to remember, what to compress, and how to retrieve it efficiently while maintaining a coherent identity.
