Reducing the time and memory complexity of traditional transformer models caused by its self-attention mechanism with long input sequences requires fundamental changes to the model architecture.  The paper "Efficient Attention Mechanisms for Large Language Models: A Survey" proposes two different solutions to this problem ( https://arxiv.org/html/2507.19595v1#S1). 
 
 
The first is "Linear Attention" in which the core computation is changed from a quadratic dot-product to a linear similarity function that leverages the associative property of matrix multiplication to avoid materializing the full attention matrix. Instead of computing all pairwise interactions explicitly, these methods rearrange the computation to achieve O(n) complexity.  The paper identifies three major paradigms within linear attention. Kernelized linear attention approximates the SoftMax operation with kernel feature maps that decompose the attention computation, using either fixed positive mappings ( ex. Linear Transformer's ELU activation) or random feature approximations (ex. Random Feature Attention). Recurrent linear attention with forgetting mechanisms introduces position-aware recurrent through data-independent decay or data-dependent gating, enabling models to control how past information fades over time. Furthermore, fast-weight and meta-learning formulations reinterpret linear attention as a memory update process with online optimization dynamics. The paper also examines hardware-efficient representations of linear attention- such as parallel, recurrent, and chunk wise forms- each offering trade-offs in computational complexity and compatibility with training or inference workflows. 
 
The second approach to addressing the complexity bottleneck is Sparse Attention, which maintains the core SoftMax mechanism but restricts computation to selected subsets of tokens. Fixed-pattern sparsity adopts static token-level masks such as sliding windows, dilated positions, or designated global tokens (ex. Big Bird), offering hardware-friendliness. Block sparsity selects or routes attention at block granularity via heuristic scoring or trainable gating, enabling structured memory access. Clustering-based sparsity organizes key-value pairs using content-based methods like k-means or locality-sensitive hashing (e.g., Reformer), facilitating semantically aware retrieval. These approaches differ in sparsity granularity, selection mechanism, and alignment with hardware. 
 
Both of these approaches transform self-attention from a computational bottleneck into a scalable mechanism. By doing so, they enable LLMs to leverage longer contexts for more informed and coherent generation, all while keeping costs during raining and inference manageable.
